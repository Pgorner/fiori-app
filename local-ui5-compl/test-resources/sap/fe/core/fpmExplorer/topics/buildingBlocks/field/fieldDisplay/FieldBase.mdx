## metaPath and contextPath

To determine what should be used in the field building block, we use its metaPath and contextPath. This information determines the target object for the field building block.
The metaPath is mandatory and is relative to the contextPath.
The contextPath, if not filled, relates to the context of the page.

```xml
<macros:Field metaPath="StringProperty" readOnly="true" id="StringProperty" />
```

<BuildingBlockPlayground binding="/RootEntity('1')">
	{`<macros:Field xmlns:macros="sap.fe.macros" metaPath="StringProperty" readOnly="true" id="StringProperty" />
					`}
</BuildingBlockPlayground>

You can define a field using a 1:1 navigation

```xml
<macros:Field metaPath="_OneToOne/Description" readOnly="true" id="OneToOne" />
```

<BuildingBlockPlayground binding="/RootEntity('1')">

					`}
</BuildingBlockPlayground>
&nbsp;

The metaPath can point to one of the following:

-   a property
-   a DataField
-   a DataPoint annotation

To Point to a DataField we need to point to an annotation like a FieldGroup with its Qualifier and its Data aggregation.

```xml
<macros:Field metaPath="@com.sap.vocabularies.UI.v1.FieldGroup#Default/Data/0" readOnly="true" id="annotation" />
```

<BuildingBlockPlayground binding="/RootEntity('1')">
	{`<macros:Field xmlns:macros="sap.fe.macros"  metaPath="@com.sap.vocabularies.UI.v1.FieldGroup#Default/Data/0" readOnly="true" id="annotation" />
					`}
</BuildingBlockPlayground>

When do we need to use a contextPath?

Let's break down the example below:

-   The controls within the cells have a binding context that is different from the binding context of the page
-   To be able to use a field building block in the ColumnListItem, an explicit contextPath is required

```xml
<m:Table items="{/ChildEntity}">
	<m:columns>
		<m:Column>
			<m:Text text="description" />
		</m:Column>
	</m:columns>
	<m:items>
		<m:ColumnListItem vAlign="Middle">
			<m:cells>
				<macros:Field metaPath="Description" contextPath="/ChildEntity" readOnly="true" id="ChildDescription" />
			</m:cells>
		</m:ColumnListItem>
	</m:items>
</m:Table>
```

<BuildingBlockPlayground binding="/RootEntity('1')">

		</m:Column>

			<m:cells>

## Formatting with Types

The field building block manages the formatting of the value according to the type. For instance, here we instantiate property 'TimeProperty' that comes with type Edm.TimeOfDay:

```xml
<macros:Field metaPath="TimeProperty" readOnly="true" id="TimeProperty" />
```

<BuildingBlockPlayground binding="/RootEntity('1')">
	{`<macros:Field xmlns:macros="sap.fe.macros" metaPath="TimeProperty" readOnly="true" id="TimeProperty" />`}
</BuildingBlockPlayground>

## Text and TextArrangement

The field building block manages the Text and TextArrangement annotation if the property or the DataField value used in its metapath is annotated with it.

```xml
<m:VBox>
	<macros:Field
		xmlns:macros="sap.fe.macros"
		metaPath="PropertyWithTextAndTextArrangementTextOnly"
		readOnly="true"
		id="TextArrangementTextOnlyProperty"
	/>
	<macros:Field xmlns:macros="sap.fe.macros" metaPath="PropertyWithText" readOnly="true" id="PropertyWithText" />
</m:VBox>
```

<BuildingBlockPlayground binding="/RootEntity('1')">
	{`<m:VBox>
<macros:Field xmlns:macros="sap.fe.macros"
